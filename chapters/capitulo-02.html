Introducción

¿Qué es ROS 2?

ROS 2 (Robot Operating System 2) es un framework de código abierto para el desarrollo de software robótico. No es un sistema operativo tradicional, sino un conjunto de bibliotecas, herramientas y convenciones que simplifican la creación de aplicaciones robóticas complejas y robustas. ROS 2 fue diseñado desde cero para superar las limitaciones de ROS 1, incorporando soporte nativo para sistemas en tiempo real, comunicaciones seguras mediante DDS (Data Distribution Service) y compatibilidad multiplataforma.

La versión Jazzy Jalisco, lanzada el 23 de mayo de 2024, es una distribución LTS (Long Term Support) con soporte hasta mayo de 2029. Está basada en Ubuntu 24.04 LTS (Noble Numbat) y representa la opción más estable y recomendada para proyectos de producción y formación académica.

¿Por qué Ubuntu?

Ubuntu es la plataforma oficialmente soportada por Open Robotics para ROS 2. Cada distribución de ROS 2 se vincula con una versión específica de Ubuntu: Jazzy Jalisco requiere Ubuntu 24.04 LTS. Instalar ROS 2 sobre Ubuntu garantiza compatibilidad completa con todos los paquetes binarios, evita problemas de dependencias y proporciona acceso a la documentación más extensa.

¿Por qué WSL2?

WSL2 (Windows Subsystem for Linux 2) permite ejecutar una distribución de Linux directamente sobre Windows sin necesidad de una máquina virtual tradicional. WSL2 utiliza un kernel Linux real proporcionado por Microsoft, ejecutado sobre el hipervisor Hyper-V. Esto significa compatibilidad total con las llamadas al sistema Linux, rendimiento de E/S cercano al nativo y, desde WSL 2.0, soporte gráfico nativo mediante WSLg para ejecutar aplicaciones como RViz2 y Gazebo directamente en el escritorio de Windows.

Paso 1

Instalar WSL2

¿Qué es WSL2?

WSL2 (Windows Subsystem for Linux 2) es la segunda generación del subsistema de Windows para Linux. A diferencia de WSL1, que traducía las llamadas al sistema Linux a llamadas NT, WSL2 ejecuta un kernel Linux real dentro de una máquina virtual ligera gestionada por Hyper-V. Esto proporciona compatibilidad completa, rendimiento de E/S significativamente superior y la capacidad de ejecutar cualquier binario ELF de Linux.

Instalación

Abra PowerShell como Administrador. Para ello, haga clic derecho sobre el botón Inicio de Windows y seleccione "Terminal (Administrador)".

Ejecute el siguiente comando:

powershell (admin) 
wsl --install

Este comando realiza automáticamente las siguientes acciones:

  • Habilita la característica opcional Virtual Machine Platform
  • Habilita la característica opcional Windows Subsystem for Linux
  • Descarga e instala el kernel Linux más reciente para WSL2
  • Establece WSL2 como la versión predeterminada
  • Descarga e instala Ubuntu como la distribución predeterminada
Reinicio obligatorio

Después de ejecutar wsl --install, Windows solicitará un reinicio del sistema. Este reinicio es obligatorio para que los componentes de virtualización se activen correctamente. No omita este paso.

Verificar la instalación

Una vez reiniciado el sistema, abra nuevamente PowerShell y ejecute:

powershell 
wsl --version

La salida mostrará información similar a:

Paso 2

Instalar Ubuntu 24.04

Desde Microsoft Store

Si el comando wsl --install no instaló Ubuntu automáticamente, puede hacerlo manualmente. Abra Microsoft Store, busque "Ubuntu 24.04 LTS" y haga clic en "Obtener".

Desde PowerShell

También puede instalarla directamente desde PowerShell:

powershell 
wsl --install -d Ubuntu-24.04

Crear usuario y contraseña

La primera vez que inicie Ubuntu, el sistema le solicitará crear un nombre de usuario y una contraseña. Este usuario será su cuenta principal dentro del entorno Linux y tendrá permisos sudo (superusuario) por defecto.

Consejo

Utilice un nombre de usuario corto y en minúsculas, sin espacios ni caracteres especiales. Por ejemplo: robotica, ros, dev. Este nombre aparecerá en el prompt de la terminal y en las rutas del sistema.

Paso 3

Actualizar Ubuntu

Antes de instalar cualquier software adicional, actualice todos los paquetes del sistema a sus últimas versiones. Ejecute los siguientes comandos uno por uno:

bash 
sudo apt update

Descripción: Actualiza la lista de paquetes disponibles desde los repositorios configurados. No instala nada, solo descarga los índices más recientes.

bash 
sudo apt upgrade -y

Descripción: Instala las versiones más recientes de todos los paquetes instalados. La opción -y acepta automáticamente la confirmación.

bash 
sudo apt autoremove -y

Descripción: Elimina paquetes que fueron instalados como dependencias y ya no son necesarios.

Buena práctica

Ejecute sudo apt update && sudo apt upgrade -y regularmente (al menos una vez por semana) para mantener su sistema seguro y estable.

Paso 4

Verificar conectividad a Internet

Antes de continuar, confirme que Ubuntu tiene acceso a Internet. Este paso es crítico porque todos los pasos siguientes requieren descargar paquetes de los repositorios.

bash 
ping -c 4 google.com

Descripción: Envía 4 paquetes ICMP al servidor DNS de Google para verificar la conectividad de red. La opción -c 4 limita a 4 intentos.

Salida esperada:

salida

64 bytes from 142.250.78.206: icmp_seq=1 ttl=115 time=12.3 ms
64 bytes from 142.250.78.206: icmp_seq=2 ttl=115 time=11.8 ms
64 bytes from 142.250.78.206: icmp_seq=3 ttl=115 time=12.1 ms
64 bytes from 142.250.78.206: icmp_seq=4 ttl=115 time=11.9 ms

Si ve 0% packet loss y tiempos de respuesta en milisegundos, la conectividad es correcta.

Si el ping falla

Posibles causas y soluciones:

  • DNS no configurado: Verifique cat /etc/resolv.conf. Debe contener un servidor DNS (por ejemplo, nameserver 8.8.8.8).
  • Firewall de Windows bloqueando: Desactive temporalmente el firewall o agregue una excepción para WSL.
  • Red corporativa con proxy: Configure las variables http_proxy y https_proxy en el entorno.
  • WSL sin red: Ejecute wsl --shutdown desde PowerShell y reinicie Ubuntu.

Paso 7

Verificar Python

ROS 2 Jazzy requiere Python 3.12, que viene preinstalado en Ubuntu 24.04.

bash 
python3 --version

Salida esperada:

salida 
Python 3.12.3

Ubuntu 24.04 LTS incluye Python 3.12 de forma predeterminada, que es la versión exacta requerida por ROS 2 Jazzy. No es necesario instalar Python manualmente.

¿Por qué Python 3.12?

Cada distribución de ROS 2 se vincula con una versión específica de Python para garantizar la estabilidad de los bindings. Jazzy fue compilado y probado contra CPython 3.12. Usar una versión diferente puede causar errores de importación en los módulos rclpy y rclcpp.

Paso 8

Instalar herramientas básicas del sistema

En este paso instalaremos únicamente las herramientas básicas del sistema operativo. Estas son dependencias necesarias para agregar repositorios, descargar archivos, compilar software y gestionar código fuente durante todo el curso.

Alcance de este paso

Este paso instala solo herramientas genéricas de Ubuntu. Las herramientas específicas de ROS 2 (colcon, rosdep, vcstool) NO se instalan aquí porque aún no hemos agregado el repositorio oficial de ROS 2. Su instalación se realiza en el Paso 13, después de instalar ros-jazzy-desktop.

¿Qué hacer si un paquete no se encuentra?

Si al ejecutar el comando de instalación aparece el error E: Unable to locate package <nombre>, significa que apt no tiene un índice actualizado que contenga ese paquete. Siga el siguiente procedimiento de diagnóstico.

Error típico

salida de error 
Reading package lists... Done
Building dependency tree... Done
Reading state information... Done
E: Unable to locate package python3-colcon-common-extensions
E: Unable to locate package python3-rosdep
E: Unable to locate package python3-vcstool

Procedimiento de solución

Paso 1. Actualice el índice de paquetes:

bash 
sudo apt update

Paso 2. Verifique que el repositorio universe esté habilitado. Si no lo está, agréguelo:

bash 
sudo add-apt-repository universe
sudo apt update

Paso 3. Busque el paquete en los repositorios disponibles:

bash 
apt search nombre_del_paquete

Por ejemplo: apt search colcon. Si la salida muestra resultados, el paquete existe y puede instalarlo. Si no aparece nada, confirme que el repositorio correcto está habilitado.

Paso 4. Identifique si el paquete pertenece al ecosistema de ROS 2.

PaqueteOrigen¿Aparece en este paso?
python3-colcon-common-extensions Repositorio oficial de ROS 2 (packages.ros.org) ❌ No todavía. Se instala en el Paso 13.
python3-rosdep Repositorio oficial de ROS 2 ❌ No todavía. Se instala en el Paso 13.
python3-vcstool Repositorio oficial de ROS 2 ❌ No todavía. Se instala en el Paso 13.
git, curl, wget Repositorios main / universe de Ubuntu ✅ Debe instalarse en este paso.
build-essential, tree Repositorio main de Ubuntu ✅ Debe instalarse en este paso.
Si los paquetes faltantes son de ROS 2, es normal

Los paquetes python3-colcon-common-extensions, python3-rosdep y python3-vcstool aún no existen en este punto del procedimiento porque el repositorio oficial de ROS 2 todavía no ha sido agregado (se hace en el Paso 11). No intente instalarlos ahora. Continúe con los pasos 9 a 12 y luego instale las herramientas de ROS en el Paso 13.

Nota técnica sobre esta corrección

Esta sección fue modificada durante la validación del libro porque, en una instalación completamente limpia de Ubuntu 24.04 LTS sobre WSL2, se detectó que los paquetes python3-colcon-common-extensions, python3-rosdep y python3-vcstool devuelven E: Unable to locate package hasta que el repositorio oficial de ROS 2 está correctamente configurado. La instalación de estas herramientas específicas de ROS se realiza en el Paso 13, después de instalar ros-jazzy-desktop, garantizando la trazabilidad completa y reproducible del entorno.

Paso 9

Configurar el Locale

ROS 2 requiere que el locale del sistema esté configurado en UTF-8 para el correcto funcionamiento del middleware DDS.

Verificar el locale actual

bash 
locale

Si el resultado muestra LANG=en_US.UTF-8 o LANG=C.UTF-8, la configuración es correcta. Si muestra POSIX o no incluye UTF-8, ejecute:

bash 
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

¿Por qué ROS 2 requiere UTF-8?

El middleware DDS de ROS 2 utiliza UTF-8 como codificación estándar para mensajes. Los nombres de tópicos, servicios, parámetros y mensajes deben estar codificados en UTF-8 para garantizar interoperabilidad entre nodos escritos en diferentes lenguajes (Python, C++).

Paso 11

Agregar el repositorio oficial de ROS 2

Descargar la llave GPG oficial

La llave GPG permite a apt verificar la autenticidad e integridad de los paquetes descargados del repositorio de ROS 2.

bash 
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg

Verificar que la llave se descargó correctamente:

bash 
ls -l /usr/share/keyrings/ros-archive-keyring.gpg

Agregar el repositorio al sources.list

bash 
echo "deb [signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
  http://packages.ros.org/ros2/ubuntu \
  $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | \
  sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

Actualizar el índice de paquetes

bash 
sudo apt update

Verificar el contenido del repositorio

bash 
cat /etc/apt/sources.list.d/ros2.list

Salida esperada:

salida 
deb [signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu noble main

El contenido indica:

  • deb — Repositorio de paquetes binarios.
  • signed-by=/usr/share/keyrings/ros-archive-keyring.gpg — Llave GPG que valida los paquetes.
  • http://packages.ros.org/ros2/ubuntu — URL del repositorio oficial.
  • noble — Codename de Ubuntu 24.04 (sustituido dinámicamente por la variable $UBUNTU_CODENAME).
  • main — Componente principal del repositorio.

Paso 13

Instalar herramientas de desarrollo de ROS

Ahora que el repositorio de ROS 2 está configurado y ros-jazzy-desktop está instalado, instale las herramientas de desarrollo específicas de ROS 2.

bash 
sudo apt install -y \
  python3-colcon-common-extensions \
  python3-rosdep \
  python3-vcstool

¿Por qué este paso se realiza DESPUÉS de instalar ROS?

Durante las pruebas de este curso se detectó que intentar instalar python3-colcon-common-extensions antes de configurar el repositorio de ROS 2 (Paso 11) podía producir el error E: Unable to locate package python3-colcon-common-extensions o instalar una versión genérica de Ubuntu incompatible con ROS 2 Jazzy.

La solución validada es instalar siempre ROS 2 primero (ros-jazzy-desktop) y luego las herramientas de desarrollo, de modo que apt resuelva las versiones correctas desde el repositorio oficial.

Paso 14

Configurar el entorno de ROS 2

Agregar el source al .bashrc

Para que ROS 2 esté disponible automáticamente cada vez que abra una terminal, agregue el archivo de configuración al ~/.bashrc:

bash

Use >> (append), nunca > (overwrite)

El operador >> agrega al final del archivo. Un solo > sobrescribiría todo el contenido del .bashrc, lo que podría causar problemas graves.

Paso 16

Verificar la instalación con ros2 doctor

ROS 2 incluye una herramienta de diagnóstico que revisa el estado del entorno, la red, los topics del sistema y posibles problemas.

bash 
ros2 doctor

El comando realiza verificaciones de:

  • Versión de la plataforma y del middleware DDS.
  • Conectividad de red entre los participantes de ROS 2.
  • Variables de entorno configuradas.
  • Estado de QoS y discovery.

Si todo es correcto, el reporte mostrará All n checks passed con una marca verde. Las advertencias amarillas no son críticas y pueden revisarse más adelante.

Paso 17

Probar ROS 2 con Talker y Listener

Esta es la prueba clásica de comunicación Publisher/Subscriber entre dos nodos de demostración.

Terminal 1 — Talker (Publisher)

terminal 1 
ros2 run demo_nodes_cpp talker

Salida esperada (flujo continuo):

salida 
[INFO] [1720000000.000000000] [talker]: Publishing: 'Hello World: 1'
[INFO] [1720000001.000000000] [talker]: Publishing: 'Hello World: 2'
[INFO] [1720000002.000000000] [talker]: Publishing: 'Hello World: 3'

Terminal 2 — Listener (Subscriber)

Abra una segunda terminal de Ubuntu y ejecute:

terminal 2 
ros2 run demo_nodes_py listener

Salida esperada:

salida 
[INFO] [1720000002.100000000] [listener]: I heard: [Hello World: 3]
[INFO] [1720000003.100000000] [listener]: I heard: [Hello World: 4]
[INFO] [1720000004.100000000] [listener]: I heard: [Hello World: 5]

Para detener los nodos, presione Ctrl+C en cada terminal.

¿Qué demuestra esta prueba?

El talker está escrito en C++ y el listener en Python. La comunicación entre lenguajes es transparente gracias al middleware DDS. Este patrón Publish/Subscribe es el modelo fundamental de ROS 2.

Paso 18

Verificar topics, nodos, interfaces y servicios

Con el talker y el listener aún en ejecución, abra una tercera terminal y ejecute los siguientes comandos de introspección.

Listar topics activos

bash 
ros2 topic list

Salida esperada: /chatter, /parameter_events, /rosout.

Echo de un topic

bash 
ros2 topic echo /chatter

Muestra los mensajes publicados en tiempo real.

Listar nodos activos

bash 
ros2 node list

Salida esperada: /talker y /listener.

Listar interfaces disponibles

bash 
ros2 interface list | head

Listar servicios activos

bash 
ros2 service list

Salida esperada: /talker/describe_parameters, /listener/describe_parameters, entre otros.

Resumen de comandos de introspección
  • ros2 topic — Tópicos (canales de datos).
  • ros2 node — Nodos (procesos de ROS 2).
  • ros2 service — Servicios (request/response sincrónico).
  • ros2 interface — Mensajes, servicios y acciones definidos.
Paso 19

Crear un Workspace de ROS 2

Un workspace es un directorio donde se compilan y organizan los paquetes de ROS 2.

bash 
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws
colcon build

Descripción:

  • mkdir -p ~/ros2_ws/src — Crea el workspace y el subdirectorio src para el código fuente.
  • cd ~/ros2_ws — Entra al workspace.
  • colcon build — Compila el workspace (estará vacío, pero genera la estructura base).

Agregar el workspace al .bashrc

bash 
echo "source ~/ros2_ws/install/setup.bash" >> ~/.bashrc
source ~/.bashrc

Verificar la estructura

bash 
tree -L 2 ~/ros2_ws

Salida esperada:

salida 
/home/usuario/ros2_ws
├── build
├── install
├── log
└── src
DirectorioDescripción
src/Código fuente. Aquí se colocan los paquetes. Es el único directorio que el desarrollador modifica.
build/Artefactos de compilación. Archivos intermedios. Se puede eliminar y regenerar.
install/Resultado de la instalación. Contiene los ejecutables y archivos de configuración. El install/setup.bash hace que los paquetes estén disponibles para ros2 run.
log/Registros de compilación. Útil para depurar errores.

Paso 21

Ejecutar RViz2

RViz2 (ROS Visualization 2) es la herramienta principal de visualización de ROS 2. Permite ver datos de sensores, modelos de robots (URDF), mapas, trayectorias, nubes de puntos y otros tipos de datos robóticos en un entorno 3D interactivo.

Paso 23

Ejecutar Gazebo Harmonic

Problema encontrado: gz sim — command not found

En esta instalación, al intentar ejecutar Gazebo directamente con el comando gz sim, se obtuvo:

$ gz sim
gz: command not found

Esto ocurre porque la instalación a través de los paquetes ros-jazzy-ros-gz-* no siempre coloca el ejecutable gz en el PATH del sistema. El binario está empaquetado como vendor y su ruta no se exporta globalmente.

Solución validada

La forma correcta y recomendada de ejecutar Gazebo Harmonic es a través de ROS 2 launch:

bash 
ros2 launch ros_gz_sim gz_sim.launch.py

Para abrir un mundo de ejemplo con formas geométricas y física activa:

bash 
ros2 launch ros_gz_sim gz_sim.launch.py gz_args:="shapes.sdf"
Primer inicio

La primera ejecución puede tardar varios minutos mientras descarga modelos y texturas. Las ejecuciones posteriores serán significativamente más rápidas.

Buenas prácticas

Actualice frecuentemente

Ejecute sudo apt update && sudo apt upgrade -y al menos una vez por semana.

No mezcle distribuciones de ROS 2

Solo haga source de una distribución a la vez en su .bashrc.
